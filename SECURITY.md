# Security Policy

## Reporting a Vulnerability

We take security seriously. If you discover a security vulnerability, please report it privately.

**Do not report security vulnerabilities via public GitHub issues.**

Instead, email: **security@opens2p.com**

You should receive a response within 48 hours. If not, please follow up.

## What to Include

- Description of the vulnerability
- Steps to reproduce
- Affected versions
- Any potential mitigations

## Scope

- The OpenS2P backend API (Python/FastAPI)
- The OpenS2P frontend (React/TypeScript)
- Docker deployment configurations
- Authentication and authorization mechanisms

## Out of Scope

- Third-party dependencies (report to respective maintainers)
- Issues requiring physical access to infrastructure

## Process

1. Report received and acknowledged within 48 hours
2. Investigation and verification
3. Fix developed and tested
4. Security advisory published
5. Public disclosure after fix is available

## Supported Versions

| Version | Supported |
|---|---|
| 0.1.x | ✅ |

## Recognition

We thank security researchers who responsibly disclose vulnerabilities.
