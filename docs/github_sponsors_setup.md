# GitHub Sponsors — exact setup steps

I can't create this account for you — it requires your own identity verification and bank details on GitHub's side. Here's exactly what to do (~15–20 min; GitHub's review can take a few days).

## 1. Decide who sponsors go to
- **The `opens2p` organization** (recommended — matches the sponsor link already on the website and in this repo's README/`.github/FUNDING.yml`: `github.com/sponsors/opens2p`)
- Your personal GitHub account

Org-level keeps funds separate from your personal account and looks more like a real open-source org.

## 2. Enable Sponsors for the org
1. Go to `github.com/opens2p` (the organization page).
2. Settings → **Sponsors**.
3. Click **Set up GitHub Sponsors** / **Join the waitlist** if prompted.

## 3. Complete the application
GitHub will ask for:
- **Country/region** and **payout method** (bank account or Stripe, depending on country).
- **Tax info** — W-9 (US) or W-8BEN (non-US individuals) via Stripe Connect.
- **A short profile** — reuse the sponsor copy already on opens2p.org.
- **Sponsorship tiers** — keep it simple at launch, e.g.:
  - $5/mo — Supporter
  - $25/mo — Backer (name listed in README)
  - $100/mo — Sponsor (logo on website)
  - $500 one-time — Founding Sponsor

## 4. Wait for approval
GitHub reviews sponsor applications. You'll get an email when approved.

## 5. Once approved
- Your page goes live at `github.com/sponsors/opens2p` — already the exact URL used everywhere (website, README, `.github/FUNDING.yml`), so nothing else needs to change.
- The **Sponsor** button will now work automatically on this repo (enabled via `.github/FUNDING.yml`, already added).

## 6. Let me know when it's live
I'll update the README/site copy if anything needs to change (e.g. real tier names, or a "Sponsors" section listing backers).
